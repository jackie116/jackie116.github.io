Hugo筆記2-資料夾結構

  2021-05-29 Word Count: 767 words Read Time: 2 minutes

上篇Hugo筆記1-介紹和安裝，介紹完基本的安裝和機制，今天來講Hugo的資料夾結構。 回想一下我們建立網站資料夾的指令：

hugo new site quickstart

我們創建名為quickstart的網站資料夾，hugo同時也在資料夾內幫忙建立好以下結構：

.
+-- archetypes  
+-- config.toml  
+-- content
+-- data
+-- layouts
+-- static
+-- themes

  • archetypes
    放文章模板的資料夾，預設會有title、date、draft: true屬性，
    可以事先設定好模板屬性，避免每次hugo new新文章都要設定。

  • config.toml
    Hugo設定檔，預設.toml，也支援.json、.yaml、.yml。
    Hugo Themes下載的主題大部分都有預設的config，可以直接替換使用。

  • content
    所有的內容(頁面)都存放在這個資料夾，資料夾內每個top-level資料夾都視為一個content section。
    Hugo預設指派content section 為content type。
    這部分看官方原文比較好理解：

Content section : A Section is a collection of pages that gets defined based on the organization structure under the content/ directory.**

Content type : A content type is a way to organize your content. Hugo resolves the content type from either the type in front matter or, if not set, the first directory in the file path. E.g. content/blog/my-first-event.md will be of type blog if no type set.

  • data
    用來存放生成網站的設定文件，或是自定義 data templates 使Hugo能動態抓取資料。

  • layouts
    存放用於生成靜態網站的.html模板。
    下載的theme都有各自的layouts，自行設計才會用到。

  • static
    存放靜態資源檔案，圖片、css、javascript

  • themes
    存放下載的主題。
    有多個主題時，網站生成時能用 -t參數指定。

其他資料夾

以上結構是網站生成時預設的資料夾，這邊再講兩個進階使用時需手動建立的資料夾：

  • assets
    存放所有需要Hugo Pipes處理的文件，只有使用了.Permalink、.RelPermalink的文件會被放到public資料夾。

  • resources
    緩存部分文件以加速網站生成，模板作者也可用來分發SASS文件，因此你不用裝預處理器。
